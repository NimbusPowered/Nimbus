Nimbus exposes a REST API (v0.2) built on Ktor. The API is optional and can be enabled in `config/nimbus.toml` or started at runtime with the `api start` console command.

## Authentication

All endpoints except `/api/health` require a Bearer token:

```http title="Request"
Authorization: Bearer <your-token>
```

Configure the token in `config/nimbus.toml`:

```toml title="config/nimbus.toml"
[api]
enabled = true
bind = "0.0.0.0"
port = 8080
token = "your-secret-token"
```

<Callout title="Auto-generated token" type="info">
If no token is configured in `nimbus.toml`, Nimbus **auto-generates** a random token on each start and logs the first 8 characters to the console. The API is never unauthenticated. Set a permanent token in the config for production use.
</Callout>

### Token Scopes

Nimbus uses two token levels:

| Token | Scope | Recipients |
|---|---|---|
| **Admin token** | Full API access (all endpoints) | Proxy/Bridge plugin, manual API calls |
| **Service token** | Service-level endpoints only (services, permissions, groups, displays, proxy sync, maintenance, metrics, network) | Game servers (SDK, Perms plugin) |

The service token is automatically derived from the admin token using HMAC-SHA256. Game servers receive the restricted service token via the `NIMBUS_API_TOKEN` environment variable, while the Velocity proxy receives the full admin token.

Admin-only endpoints (require the admin token): `/api/reload`, `/api/stress/*`, `/api/files/*`, `/api/config`, `/api/loadbalancer`, `/api/nodes`.

<Callout title="Rate limiting" type="info">
The API enforces **per-IP** rate limits: **120 requests/minute** globally and **5 requests/minute** for stress test endpoints. Exceeding the limit returns HTTP 429 (Too Many Requests).
</Callout>

### Token Management

JWT-based scoped tokens are available as an opt-in feature. Enable with `jwt_enabled = true` in the `[api]` section of `nimbus.toml`. Legacy bearer tokens always work regardless of this setting.

#### POST /api/tokens

Generate a scoped JWT token. **Admin-only.**

**Request Body:**

```json title="Request"
{
  "subject": "my-panel",
  "scopes": ["services:read"],
  "expiresInSeconds": 3600
}
```

**Response:**

```json title="Response"
{
  "token": "eyJhbGciOiJIUzI1NiIs..."
}
```

| Status Code | Description |
|-------------|-------------|
| 200 | Token generated |
| 400 | Invalid scopes or parameters |
| 403 | JWT not enabled or not admin |

#### GET /api/tokens/scopes

List all available permission scopes. **Admin-only.**

**Response:**

```json title="Response"
{
  "scopes": [
    "services:read",
    "services:write",
    "groups:read",
    "groups:write",
    "metrics:read",
    "cluster:read",
    "cluster:write",
    "files:read",
    "files:write",
    "config:read",
    "config:write",
    "audit:read",
    "admin"
  ]
}
```

| Status Code | Description |
|-------------|-------------|
| 200 | Success |

<Callout type="info">
The `admin` scope is a superscope that grants access to all endpoints. Other scopes follow a `resource:action` pattern. JWT tokens are validated on every request and rejected if expired or if the requested endpoint is outside the token's scopes.
</Callout>

## Response Format

All responses use JSON. Error responses follow a consistent format:

```json title="Error Response"
{
  "success": false,
  "message": "Description of what went wrong",
  "error": "ERROR_CODE"
}
```

The `error` field contains a machine-readable error code that clients can switch on instead of parsing messages. For successful responses, the `error` field is `null`.

**Common error codes:**

| Error Code | Description |
|------------|-------------|
| `SERVICE_NOT_FOUND` | The requested service does not exist |
| `SERVICE_NOT_READY` | Service exists but is not in READY state |
| `SERVICE_START_FAILED` | Failed to start service (max instances or JAR unavailable) |
| `GROUP_NOT_FOUND` | The requested group does not exist |
| `GROUP_ALREADY_EXISTS` | A group with that name already exists |
| `GROUP_HAS_RUNNING_INSTANCES` | Cannot delete group with running services |
| `VALIDATION_FAILED` | Request failed input validation |
| `INVALID_INPUT` | Invalid parameter value |
| `NOT_FOUND` | Generic resource not found |
| `FORBIDDEN` | Action not allowed |
| `READ_ONLY` | Scope is read-only |
| `PATH_TRAVERSAL` | Path traversal attempt blocked |
| `INTERNAL_ERROR` | Unexpected server error |
| `STRESS_ALREADY_RUNNING` | A stress test is already active |
| `STRESS_NOT_RUNNING` | No stress test is currently active |
| `CLUSTER_NOT_ENABLED` | Cluster mode is not enabled |
| `LOAD_BALANCER_NOT_ENABLED` | Load balancer is not enabled |
| `PROXY_NOT_AVAILABLE` | No Velocity proxy available |
| `COMMAND_NOT_FOUND` | Command does not exist |
| `COMMAND_NOT_REMOTE` | Command does not support remote execution |

## Services

### GET /api/services

List all services. Supports filtering by group, state, and custom state.

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `group` | string | Filter by group name |
| `state` | string | Filter by state: `PREPARING`, `STARTING`, `READY`, `STOPPING`, `STOPPED`, `CRASHED` |
| `customState` | string | Filter by custom state (case-insensitive) |

**Example:**

```bash title="Terminal"
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8080/api/services?group=Lobby&state=READY"
```

### POST /api/groups

Create a new server group. Writes a TOML config file and reloads.

**Request Body:**

```json title="Request Body"
{
  "name": "SkyWars",
  "type": "DYNAMIC",
  "template": "skywars",
  "software": "PAPER",
  "version": "1.21.4",
  "memory": "2G",
  "maxPlayers": 16,
  "minInstances": 1,
  "maxInstances": 8,
  "playersPerInstance": 16,
  "scaleThreshold": 0.8,
  "idleTimeout": 300,
  "stopOnEmpty": true,
  "restartOnCrash": true,
  "maxRestarts": 5,
  "jvmArgs": ["-XX:+UseG1GC"]
}
```

All fields except `name` and `template` have defaults. Optional fields:

| Field | Default | Description |
|-------|---------|-------------|
| `type` | `DYNAMIC` | `DYNAMIC` or `STATIC` |
| `software` | `PAPER` | `PAPER`, `PURPUR`, `VELOCITY`, `FORGE`, `NEOFORGE`, `FABRIC`, `CUSTOM` |
| `version` | `1.21.4` | Minecraft version |
| `modloaderVersion` | `""` | Modloader version (Forge/NeoForge/Fabric) |
| `jarName` | `""` | Custom JAR filename |
| `readyPattern` | `""` | Custom regex for ready detection |
| `memory` | `1G` | Memory per instance |
| `maxPlayers` | `50` | Max players per instance |
| `minInstances` | `1` | Minimum running instances |
| `maxInstances` | `4` | Maximum running instances |
| `playersPerInstance` | `40` | Players per instance (for scaling) |
| `scaleThreshold` | `0.8` | Scale up when this % full (0.0-1.0) |
| `idleTimeout` | `0` | Stop idle instances after this many seconds (0 = never) |
| `stopOnEmpty` | `false` | Stop when no players |
| `restartOnCrash` | `true` | Auto-restart on crash |
| `maxRestarts` | `5` | Max restart attempts |
| `jvmArgs` | `["-XX:+UseG1GC"]` | JVM arguments |

**Validation Rules:**
- Names must match `^[a-zA-Z0-9_-]{1,64}$`
- Memory format: `512M` or `2G`
- Version format: `1.21.4` or `1.21.4-pre1`
- `minInstances <= maxInstances`
- `scaleThreshold` between 0.0 and 1.0

| Status Code | Description |
|-------------|-------------|
| 201 | Group created |
| 400 | Validation error |
| 409 | Group already exists |

### POST /api/broadcast

Broadcast a message to all running services, or to a specific group.

**Request Body:**

```json title="Request Body"
{
  "message": "Server restarting in 5 minutes!",
  "group": "Lobby"
}
```

The `group` field is optional. When omitted, the message is sent to all READY services.

Backend servers receive the message via `/say`, proxy servers via `velocity broadcast`.

**Response:**

```json title="Response"
{
  "success": true,
  "message": "Broadcast sent to 4/4 services in network",
  "services": 4
}
```

**Example:**

```bash title="Terminal"
curl -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"message": "Maintenance in 10 minutes!"}' \
  http://localhost:8080/api/broadcast
```

| Status Code | Description |
|-------------|-------------|
| 200 | Broadcast sent (check `services` count in response) |
| 404 | No ready services found in the target scope |

List a directory or read a file. Directories return a file listing. Text files return content as JSON. Binary files (`.jar`, `.zip`, `.png`, etc.) are served as downloads.

**Response:**

```json title="Response (200 OK)"
{
  "name": "Adrenaserver",
  "version": "1.5.2",
  "mcVersion": "1.21.4",
  "modloader": "FABRIC",
  "modloaderVersion": "0.16.9",
  "totalFiles": 59,
  "serverFiles": 47,
  "source": "MODRINTH"
}
```

The `source` field indicates the modpack type: `MODRINTH`, `CURSEFORGE_API`, or `SERVER_PACK`.

### POST /api/modpacks/import

Import a modpack as a new server group. Accepts Modrinth slugs/URLs, CurseForge slugs/URLs (`curseforge:slug`), or local file paths.

**Auth:** Admin token required.

**Request:**

```json title="Request"
{
  "source": "adrenaserver",
  "groupName": "Modded",
  "type": "DYNAMIC",
  "memory": "2G",
  "minInstances": 1,
  "maxInstances": 2
}
```

**Response:**

```json title="Response (201 Created)"
{
  "success": true,
  "message": "Modpack 'Adrenaserver' imported as group 'Modded'",
  "groupName": "Modded",
  "filesDownloaded": 47,
  "filesFailed": 0
}
```

### POST /api/modpacks/upload

Upload a server pack ZIP or `.mrpack` file and import it as a new group. The file is streamed directly to disk тАФ no memory buffering.

**Auth:** Admin token required.

**Content-Type:** `application/octet-stream` (raw file body, not multipart).

**Query Parameters:**

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `groupName` | Yes | тАФ | Name for the new group |
| `type` | No | `DYNAMIC` | `STATIC` or `DYNAMIC` |
| `memory` | No | `2G` | Memory per instance |
| `minInstances` | No | `1` | Minimum instance count |
| `maxInstances` | No | `2` | Maximum instance count |
| `fileName` | No | `upload.zip` | Original filename (for detection) |

**Response:**

```json title="Response (201 Created)"
{
  "success": true,
  "message": "Server pack imported as group 'ATM10' (NEOFORGE 21.1.224, MC 1.21.1, 449 mods)",
  "groupName": "ATM10",
  "filesDownloaded": 449,
  "filesFailed": 0
}
```
